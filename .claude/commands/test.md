# /test

Verify the implementation with tests.

**Phase: VERIFY**

When invoked:

1. **Identify gaps** — What scenarios aren't covered by tests?
2. **Write tests first** — TDD: tests define behavior
3. **Implement code** — Make tests pass
4. **Verify coverage** — All happy paths and edge cases
5. **Debug failures** — If tests fail, diagnose and fix

Test types:
- Unit tests (individual functions)
- Integration tests (components working together)
- End-to-end tests (full user flows)
- Edge cases (boundary conditions, errors)

Reference the `test-driven-development` and `debugging-and-error-recovery` skills.
