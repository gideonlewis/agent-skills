# /review

Audit code quality before merging.

**Phase: REVIEW**

When invoked:

1. **Check correctness** — Does it do what it claims?
2. **Check clarity** — Is the code easy to understand?
3. **Check testing** — Are edge cases covered?
4. **Check performance** — Any obvious inefficiencies?
5. **Check security** — Any vulnerabilities?
6. **Check style** — Matches repo conventions?

Output a review with:
- ✓ What's good
- 🐛 Bugs found
- ⚡ Performance issues
- 🔒 Security concerns
- 💡 Suggestions for clarity

Reference the `code-review-and-quality`, `security-and-hardening`, and `performance-optimization` skills.
