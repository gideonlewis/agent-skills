# /spec

Start specification-driven development by capturing requirements clearly.

**Phase: DEFINE**

When invoked, help the user write a structured specification covering:

1. **Objective** — What problem are we solving? For whom?
2. **Scope** — What's in scope? What's explicitly out?
3. **Requirements** — Core features and acceptance criteria
4. **Design** — API, UI, architecture decisions
5. **Constraints** — Time, tech stack, team size
6. **Success Criteria** — How do we know it worked?

Output a `SPEC.md` in the project root with all six sections. Save only after user confirmation.

Reference the `spec-driven-development` skill for detailed guidance.
